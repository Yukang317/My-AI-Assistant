#### Step 7：`rag/indexer.py` — 文档摄入编排器（Mode E）

全部 6 个函数完成，串联 Parser → Chunker → Embedding → MinIO → Milvus 五大模块：

| 函数 | 说明 |
|------|------|
| `_compute_md5` | 计算文件 MD5（1 行） |
| `_build_object_key` | 构建 MinIO 对象路径（5 行） |
| `__init__` | 初始化 4 个子模块（4 行） |
| `index_document` ★核心 | 完整 8 步摄入管线，含 MD5 去重（new/skipped/updated） |
| `_prepare_chunks_for_milvus` | dataclass → dict 防腐层（37 行） |
| `delete_document` | MinIO + Milvus 双清（12 行） |

**核心架构概念**：
- **管道模式（Pipeline Pattern）**：8 步单向流水线，每步失败即停止
- **MD5 去重三态**：新文件 → 走管线 / 内容相同 → 跳过 / 内容更新 → 先删旧再索引
- **防腐层（ACL）**：`_prepare_chunks_for_milvus` 隔离 dataclass 和 Milvus dict 两种"方言"
- **MinIO-before-Milvus 顺序**：先存文件后写向量，MinIO 失败不污染 Milvus
- **zip() 拉链遍历**：`for chunk, vec in zip(parents, parent_vectors)` 一一配对

**本次核心讨论**：

| 问题 | 结论 |
|------|------|
| 为什么要校验向量数和文本数一致？ | embed() 契约是 N 进 N 出，但 API 模式可能跳过空字符串；fail-fast 原则，成本几乎为零 |
| 哪个阶段的什么数量一致？ | parent_texts 长度 == parent_vectors 长度（同理 child），不是跨阶段对应 |
| zip() 遍历怎么写？ | `zip(parents, parent_vectors)` 一一配对，父块 3+3 向量 → 3 轮循环，子块多一个 child_id |
