## 完整 QA 记录

### 用户 1："那我们下一步该干啥啦，你先给我介绍一下我们现在的智能体架构好不"

**Claude**：详细介绍了当前 Agent 架构全景：

```
START → load_context → intent_route（条件分支）
  ├─ general_chat → result_synthesis → END
  └─ use_tool → tool_execute ────────────┘
```

五层架构分解：
1. **State 层**（`agent/state.py`）：12 字段 TypedDict，`StateField` 枚举管理所有 key
2. **路由层**（`agent/router.py`）：LLM 看用户问题 + 可用工具列表 → 返回 JSON `{intent, target_tool, reason}`，JSON prefill 技巧防噪声输出
3. **工具系统**（`agent/tools/`）：注册表模式（TOOLS 字典），统一 `(ToolContext, str) -> ToolResult` 签名。当前只实现 `rag_search`，`web_search` 和 `document_search` 为占位符
4. **安全层**（`agent/bound.py`）：DANGER_ZONES / NEVER_DO / IRON_LAWS，deny-first 设计
5. **结果合成**（`agent/nodes/result_synthesis.py`）：所有路径汇聚点，拼装消息 → LLM 生成 → 保存 PG

当前架构局限：单轮调用、单工具、无流式输出、MemorySaver 非持久化、意图检测 prompt 简单。

### 用户 5："我想不出来啊咋办，具体的一件事就是不能随时随地和一个懂我的AI聊天，很多跟AI说话AI也只会顺着我的思路走，正常聊天都起不到安慰的效果，还有时候不想打字只想聊天用嘴巴说，这个方向只有豆包可以，但是豆包又傻得要死，只会顺着我的话说，然后翻旧账，总是说要不要听听华晨宇的歌吃点巧克力？我觉得这种虽然是我自己喜欢的事情但是它说出来让我做就尴尬的要死。让他发散找些资料或者实际的案例给我启发也找不到合适的或者符合我口味的，有时候想搞一个实时翻译的系统但是觉得又不现实。我知道如果我说这些你肯定会顺着我的思路走，但是我就是很焦虑，我想让你给我一个很新奇又实用的角度，真烦。你觉得呢？"

### Claude 提出的方向（⚠️ 待讨论/未定案）

**不做"个人 AI 助理"了。做"苏格拉底式对话伙伴"。**

两个品类的根本差异：

| | 个人 AI 助理 | 苏格拉底式对话伙伴 |
|---|---|---|
| 目标 | 帮你做事、回答问题 | 帮你想得更深 |
| 姿态 | 顺从、服务型 | 挑战、追问型 |
| 对话模式 | "好的，我帮你查一下" | "你确定吗？有没有想过另一种可能？" |
| 回忆你 | "你喜欢华晨宇" | "你上周三说到焦虑的原因，这次提到的好像不太一样" |
| 找资料 | 搜关键词返回 | 找反例、找边界 case、找你不爱听但值得看的东西 |
| 核心指标 | 任务完成率 | 你被问住了几次 |

**为什么这个方向适合当前项目**：

1. **已有全部技术基础** — RAG 检索不是为了"回答问题"，而是为了**找到和你当前观点对立的案例/研究/数据**；Agent 编排不是为了"执行任务"，而是**主动规划对话方向**；MEMORY.md 记忆不是为了"记住你喜欢什么"，而是**追踪你的思维变化轨迹**

2. **这是一个真正没人做的品类** — 豆包做不了（KPI 是用户满意度）、ChatGPT 做不了（安全策略要求避免争议）、DeepSeek 做不了（在追生产力工具赛道）。只有一个人用的东西，不需要考虑"会不会冒犯其他用户"

3. **RAG 系统天然适配这个场景** — 你说话 → Agent 自动搜"和这个观点相反的资料"；你表达情绪 → Agent 搜"历史上遇到类似处境的人是怎么走出来的"（真实案例而非鸡汤）；你觉得某个想法很对 → Agent 主动找反例

**关键洞察**：用户提到的所有痛点——AI 太顺从、推荐太尴尬、不会找符合口味的东西——**恰好是因为现有产品把"讨好用户"放在第一位**。"苏格拉底式对话伙伴"的核心设计就是**不讨好用户**。
