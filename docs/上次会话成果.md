### P1：step23_rag.py 接入 /chat/graph 端点 + 异步并发底层深度讨论（6月21日，第18次会话）

**完成内容**：
- `step23_rag.py` 已接入 `POST /chat/graph` 端点（对照 `_ref_step23_rag.py` 实现，4 处插入）
- 新增 `generate_graph_stream_response()` 函数（伪流式 SSE 生成器）
- 深入讲解 Python 异步并发底层原理（从三行代码串起整个知识体系）

**本次核心讨论**：

| 问题 | 结论 |
|------|------|
| 事件循环是什么？为什么叫"事件循环"？ | `while True` 循环 + `epoll_wait()` 等 OS 事件通知，事件=OS 通知的状态变化，循环=处理完继续等 |
| `run_in_executor` 在干什么？Future 在哪？ | 把同步阻塞函数委托给线程池执行，立即返回 Future（占位对象），await 等到完成后拿结果 |
| 线程池和事件循环是什么关系？ | 事件循环拥有线程池但不等于它，`run_in_executor` 是唯一合法委托出口，通过 Future 通信 |
| GIL 是什么？跟事件循环什么关系？ | CPython C 源码里的互斥锁，保护内存管理。I/O 等待期间自动释放，所以 asyncio 对 I/O 密集高效 |
| 为什么 yield 只执行一次？ | `run_graph` 用 `model.invoke()`（同步返回全文），不是 `model.stream()`（逐 token），data 源头是一大坨 |
| 进程 vs 线程？项目实际用多少线程？ | 一个 uvicorn 进程内部：1 主线程 + ~20 线程池线程 + 1-2 辅助线程 ≈ 22-23 个 OS 线程 |
| Nginx 缓存 vs 缓冲？no-cache 的逻辑？ | 两件独立的事——HTTP 缓存（语义层，管"能不能用副本"）vs Nginx 缓冲（传输层，管"要不要攒够再发"）|

**关键概念卡片**（11 个新概念，已写入日志）：

| 名词 | 一句话 |
|------|--------|
| 事件循环 | `while True` 等 OS 事件通知 → 分发给协程 |
| GIL | CPython C 源码互斥锁，保证同一时刻一个线程执行 Python 字节码 |
| Future | `asyncio` 占位对象，pending → 线程池完成 → done |
| StreamingResponse | FastAPI 响应类，一边 yield 一边往 TCP 写，不等全部生成 |
| SSE | HTTP 长连接单向推送，`EventSource` 接收 |
| Nginx | 反向代理，放 uvicorn 前面处理 HTTPS/限流/负载均衡 |
| proxy_buffering | Nginx 默认攒数据再发，SSE 必须关 |

**修正的理解偏差**：
- ❌ "进程就是 Python 项目文件" → ✅ 进程是文件加载到内存后跑起来的活实例
- ❌ "建议加个 GIL 锁" → ✅ GIL 是 CPython 硬编码的 C 互斥锁，用户无法选择
- ❌ "保证同一时刻只有一个线程运行 Python 代码" → ✅ 这是副作用，GIL 的目的是保护 CPython 内存管理

## 核心知识点记录

| 知识点 | 一句话 |
|--------|--------|
| RAG 两大模块职责分离 | DocumentIndexer 写入（离线）↔ RagService 读取（在线），共享底层但互不包含 |
| 懒加载 vs 直接初始化 | 保护启动流程：Milvus 挂了不影响普通聊天，首次 ~1-2s，后续 ~0ms |
| BM25 内存索引 vs 向量持久化 | BM25 重启丢失需从 Milvus 重建；向量存在 Milvus 重启后直接可用 |
| 从 Milvus 反查构建 BM25 | 单一数据源原则，避免数据不一致（生产环境通常独立持久化） |
| Python import 全局解析 | 一个 import 失败会级联瘫痪所有依赖模块 |
| etcd 在 Standalone 中的角色 | Milvus Standalone 把分布式组件压缩进一个进程，内部仍用 etcd 做元数据协调 |
| Docker Compose exec vs shell | YAML 列表格式 = exec 模式，字符串 = shell 模式（/bin/sh -c 包装） |
| Agent 循环不值钱 | ~20 行代码，98% 是 harness 基础设施（权限/压缩/编排/可观测） |
| 工具 = 函数 + TOOL.md + 注册 | 不是完整子图，更轻量 |
| BOUND 约束从第一天写 | DANGER_ZONES / NEVER_DO / IRON_LAWS，确定性代码兜底 |
| 文件记忆 vs 向量记忆 | Markdown 文件可读可改可版本控制，适合个人助理起步 |
| 适配层模式 | LangGraph 只传 state，业务函数要 model → 适配层 = get_llm() + 转发 |
