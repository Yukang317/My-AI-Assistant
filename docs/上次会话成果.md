## 核心知识点记录

| 知识点 | 一句话 |
|--------|--------|
| RAG 两大模块职责分离 | DocumentIndexer 写入（离线）↔ RagService 读取（在线），共享底层但互不包含 |
| 懒加载 vs 直接初始化 | 保护启动流程：Milvus 挂了不影响普通聊天，首次 ~1-2s，后续 ~0ms |
| BM25 内存索引 vs 向量持久化 | BM25 重启丢失需从 Milvus 重建；向量存在 Milvus 重启后直接可用 |
| 从 Milvus 反查构建 BM25 | 单一数据源原则，避免数据不一致（生产环境通常独立持久化） |
| Python import 全局解析 | 一个 import 失败会级联瘫痪所有依赖模块 |
| etcd 在 Standalone 中的角色 | Milvus Standalone 把分布式组件压缩进一个进程，内部仍用 etcd 做元数据协调 |
| Docker Compose exec vs shell | YAML 列表格式 = exec 模式，字符串 = shell 模式（/bin/sh -c 包装） |
| Agent 循环不值钱 | ~20 行代码，98% 是 harness 基础设施（权限/压缩/编排/可观测） |
| 工具 = 函数 + TOOL.md + 注册 | 不是完整子图，更轻量 |
| BOUND 约束从第一天写 | DANGER_ZONES / NEVER_DO / IRON_LAWS，确定性代码兜底 |
| 文件记忆 vs 向量记忆 | Markdown 文件可读可改可版本控制，适合个人助理起步 |
| 适配层模式 | LangGraph 只传 state，业务函数要 model → 适配层 = get_llm() + 转发 |
