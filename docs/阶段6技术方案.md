## 参考项目

以下是本阶段 Agent 设计的主要参考项目和工程方法论：

### 工程方法论

| 工程领域 | 核心关注点 | 本项目中的应用/参考 |
|---------|-----------|-------------------|
| **Prompt Engineering** | 提示词的设计、测试与优化——如何用最少 token 引导 LLM 输出结构化、准确的结果 | 意图路由提示词（`agent/router.py`）、工具调用提示词、结果合成提示词 |
| **Context Engineering** | 在推理时管理和组织 LLM 可用的信息——上下文压缩、结构化组织、动态窗口管理 | 上下文加载节点（`agent/nodes/load_context.py`）、MEMORY.md 文件记忆模式 |
| **Harness Engineering** | LLM 周围的基础设施——权限控制、执行环境安全、可观测性、编排层 | BOUND 约束层（`agent/bound.py`）、工具注册表可插拔架构 |
| **Loop Engineering** | Agent 迭代执行循环的设计——while-true 循环、ReAct 模式、退出条件与预算控制 | Agent 执行循环设计（while-true 风格，max_turns 硬限制） |

### 参考项目

| 项目 | 借鉴的设计要点 |
|------|--------------|
| **Claude Code** | Agent 循环是小的部分，Harness 是大的部分；文件系统 > 向量数据库做记忆；MEMORY.md 模式 |
| **OpenClaw** | 星型拓扑（单一入口 FastAPI）、TOOL.md 惰性加载、能力扩展 = 写文档不是写代码 |
| **Codex** | BOUND 约束层（deny-first 权限）、确定性代码守住底线、skills 机制 |
| **Harness** | 双角色分离（Router + Executor）、验证循环（执行→检查→修复） |
| **sagt_agent** | LangGraph State + StateField 枚举模式、HITL interrupt 机制、主图+子图结构 |

## 一、已实现：Agent 框架骨架（阶段 6.1）

### 1.1 实际运行的架构

```
用户请求
  │
  ▼
FastAPI :1000  /chat/graph
  │
  ▼
Main Graph（主图）
  │
  ├─ ① load_context      ← PG 历史消息 + MEMORY.md 文件
  ├─ ② intent_router      ← LLM 判断意图 → 选择目标工具
  ├─ ③ execute_tool       ← 调用注册的工具（当前仅 rag_search）
  ├─ ④ result_synthesis   ← 整合结果 + 保存消息 + 流式输出
  │
  ▼
SSE 流式响应
```

**两条执行路径**：
- `use_tool` → execute_tool → result_synthesis（工具调用路径）
- `general_chat` → result_synthesis（直接闲聊路径）

### 1.3 已实现的关键设计点

**工具注册机制（可插拔）**：
```python
# agent/tools/registry.py — 已实现
TOOLS = {
    "rag_search": {
        "func": rag_search,
        "description": "搜索个人知识库，返回相关文档片段",
        "bound": ["READ_ONLY"],
    },
}
```
新增工具 = 写一个函数 + 注册一行。

**BOUND 约束层**：
```python
# agent/bound.py — 已实现
BOUND = {
    "DANGER_ZONES": ["/etc", "~/.ssh", "*.env", "*.key"],
    "NEVER_DO": ["删除文件", "执行shell命令", "修改系统配置"],
    "IRON_LAWS": ["回复必须使用中文", "不确定时反问用户", "不编造信息"],
}
```
每次工具调用前通过确定性代码检查，不信任 LLM 输出。

**记忆文件（文件系统模式）**：
```
personal_assistant/
├── memory/
│   ├── MEMORY.md              # 用户画像 + 长期偏好（静态，人工维护）
│   ├── sessions/
│   │   └── {session_id}.md    # 单次会话记录
│   └── daily/
│       └── 2026-06-11.md      # 每日活动日志
```

## 三、与 sagt_agent 的对比（仅对比已实现部分）

| 维度 | sagt_agent（参考项目） | personal_assistant（阶段 6.1 已实现） |
|------|----------------------|-------------------------------------|
| 架构 | 主图 + 6 子图 | 主图 + 工具注册表（更轻量） |
| 扩展方式 | 每个功能是独立子图 | 每个功能注册到工具表 |
| 状态管理 | TypedState + StateField 枚举 | 同样模式（借鉴而来） |
| 意图路由 | LLM 判断 → 子图 | LLM 判断 → 工具 |
| 工具调用 | 无 while-true 循环（固定路径） | 无 while-true 循环（当前同样为固定路径） |
| HITL | 企微 interrupt 确认 | 未涉及（预留接口） |
