## 二、概念笔记

### 2.1 `uv init` 是干嘛的？

顺序是 **`uv init` 先生成 toml 文件，然后你再往里写依赖**，不是"先写好 toml 再 init"。

```bash
uv init                          # 生成空的 pyproject.toml（只有 name、version、python 要求）
uv add fastapi openai uvicorn    # 每加一个包，uv 自动更新 pyproject.toml + 生成 uv.lock
```

`uv init` 的作用就两件事：
- 创建 `pyproject.toml`（项目身份证，告诉 uv "这是一个 Python 项目"）
- 创建 `.python-version`（锁定 Python 版本）

类比：`git init` 创建 `.git` 目录，之后你才能 `git add`、`git commit`。

### 2.2 uvicorn 是什么？

**uvicorn 是一个 ASGI 服务器程序。** FastAPI 的 `@app.post`、`@app.get` 只是"规则书"（请求来了该调哪个函数），但规则书自己不会接客——uvicorn 才是站在门口、监听端口、接收 HTTP 请求、转交 FastAPI 的"前台"。

```
浏览器 → 网络请求 → uvicorn（端口上等着）→ FastAPI app（匹配路由）→ 你的函数
```

类比：

| 角色 | 类比 |
|------|------|
| `step3_api.py` 里你写的函数 | 餐厅后厨的厨师 |
| FastAPI (`app = FastAPI()`) | 菜单（哪个菜对应哪个厨师） |
| **uvicorn** | 服务员（接客、递单、端菜） |

`uvicorn.run(app, host='0.0.0.0', port=1000)` 翻译成人话："服务员，在 1000 号桌等着，任何人来都接待。"

### 2.3 localhost vs 公网 IP

项目部署在阿里云 ECS 上，测试分两层：

**第一层：ECS 上自测（用 localhost）**

```bash
# SSH 连上 ECS，在 ECS 终端里跑：
curl http://localhost:1000/chat -d '{"question":"你好"}'
```

`localhost` 是 ECS 自己指自己——请求没出服务器，相当于厨房里自己尝菜。

**第二层：从笔记本浏览器访问（用公网 IP）**

```
浏览器输入：http://<ECS公网IP>:1000/
```

请求链路：笔记本 → 互联网 → ECS 公网网卡 → uvicorn。能通靠两个条件：

| 条件 | 对应代码/配置 | 在哪 |
|------|-------------|------|
| uvicorn 监听所有网卡 | `host='0.0.0.0'` | step3_api.py |
| 阿里云安全组放行端口 | 入方向规则 | 阿里云控制台 |

> `host='0.0.0.0'` = "任何来源都接"。如果写 `host='127.0.0.1'`，只有 ECS 自己能访问。

**HTML 里的 `/chat` 为什么也能通？** `fetch("/chat")` 是相对路径，会自动拼上当前页面的域名。浏览器拿到 HTML 时已经知道自己在 `http://<公网IP>:1000/`，fetch 就自动请求 `http://<公网IP>:1000/chat`，不需要写死 IP。

```
你的笔记本浏览器                    ECS 云服务器
─────────────────                   ─────────────────
http://<IP>:1000/     ──网络──→    uvicorn (0.0.0.0:1000)
                                       │
                                       ↓
                                  FastAPI app
                                       │
                                  @app.get("/") → index() → 返回 HTML
                                  @app.post("/chat") → chat() → DeepSeek API
```

### 2.4 CORS 跨域资源共享

浏览器有**同源安全策略**：从一个来源（origin）加载的页面，默认不能请求另一个来源的数据。CORS 就是服务器主动声明"我允许跨域"。

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],     # 任何来源都接受
    allow_methods=["*"],     # GET/POST 都行
    allow_headers=["*"],     # 任何请求头都收
)
```

`add_middleware` 给 FastAPI 装一个**拦截器**：每个请求进来，CORS 中间件先检查来源，然后盖"放行"章。没有它，浏览器的 `fetch` 请求会被直接拦截。

### 2.5 HTML / CSS / JS 分工

| 语言 | 角色 | 类比 |
|------|------|------|
| **HTML** | 结构（骨架） | 盖了几堵墙、开了几扇窗 |
| **CSS** | 样式（皮囊） | 墙刷白漆、窗户装蓝色框 |
| **JavaScript** | 行为（动作） | 人走进房间摁开关、拉开窗帘 |

HTML 是**静态**的——写完什么样就永远什么样。`<div id="messages"></div>` 只是一个空壳子。JS 做的事是**往空壳子里动态填内容**：

```javascript
addMessage("你好", "user")  // 做三件事：
// 1. document.createElement("div")  — 凭空造一个 div（HTML 里没有的）
// 2. div.className = "message user" — 套上 CSS 的皮（蓝色气泡）
// 3. messagesDiv.appendChild(div)    — 把它插进页面
```

JS 核心语法对照 Python：

| Python | JavaScript |
|--------|------------|
| `def fn():` | `function fn() { }` |
| `await x` | `await x` |
| `None` | `null` |
| `dict` | `object`（类似但不完全一样） |
| `list` | `array` |
| `print()` | `console.log()` |
| `f"{x}"` | `` `模板字符串 ${x}` `` |

变量声明：`const`（常量）、`let`（可变）、`var`（旧语法，别用）。

### 2.6 `@app.get` 与 `@app.post`

**GET vs POST**：

| 方法 | 触发场景 | 类比 |
|------|---------|------|
| GET | 浏览器地址栏输入网址 | 去图书馆"给我《三体》"，管理员把书递给你 |
| POST | 提交表单、fetch 发数据 | 去邮局寄包裹，东西交给柜台处理 |

`@app.get("/")` 逐行拆解：

```python
@app.get("/", response_class=HTMLResponse)  # 注册 GET 路由，根路径 "/"
def index():                                 # 函数名无所谓，叫 index 是惯例
    with open("static/index.html", "r", encoding="utf-8") as f:
        return f.read()                      # 返回整个 HTML 给浏览器
```

| 行 | 是什么 | 为什么 |
|----|--------|--------|
| `@app.get("/")` | 注册 GET 路由 | `"/"` = 根路径，访问首页触发 |
| `response_class=HTMLResponse` | 告诉 FastAPI 返回 HTML | 不加会当 JSON 返回，浏览器显示纯文本 |
| `open(..., encoding="utf-8")` | 读文件 | 保证中文不乱码 |
| `return f.read()` | 返回文件内容 | HTML 字符串直接送给浏览器渲染 |

**对比两个端点**：

```python
# POST：你发数据我处理 → 返回 JSON
@app.post("/chat", response_model=ChatResponse)
def chat(req: ChatRequest):
    ...
    return ChatResponse(reply=reply_content)

# GET：你访问我返回页面 → 返回 HTML
@app.get("/", response_class=HTMLResponse)
def index():
    ...
    return f.read()
```

### 2.7 LLM 调用核心概念

**messages 三个角色**：

| role | 含义 | 示例 |
|------|------|------|
| `system` | 系统指令，定调 | "你是一个乐于助人的个人助理" |
| `user` | 用户说的话 | "你好，我叫张三" |
| `assistant` | 模型自己的回复 | "你好张三！有什么可以帮你的？" |

**temperature**：控制创造性，范围 0.0 ~ 2.0

| 值 | 效果 | 适用场景 |
|----|------|---------|
| 0.0 | 最保守，回复几乎一样 | 分类、提取、代码生成 |
| 0.7 | 平衡创造性和稳定性 | 日常聊天 |
| 2.0 | 最放飞，脑洞大 | 创意写作 |

**response 取值路径**：`response.choices[0].message.content`

```
response                    → ChatCompletion 对象
  .choices                  → 候选回复列表（通常就一个）
    [0]                     → 第一个候选
      .message              → 消息对象（含 role + content）
        .content            → 真正的回复文本（字符串）
```

### 3.1 step1_hello_llm.py（最终版）

```python
import os
from dotenv import load_dotenv
from openai import OpenAI

load_dotenv()
api_key = os.getenv("DEEPSEEK_API_KEY")
base_url = os.getenv("DEEPSEEK_BASE_URL")
model = "deepseek-chat"

client = OpenAI(api_key=api_key, base_url=base_url)

messages = [
    {"role": "system", "content": "你是一个乐于助人的个人助理，用中文回答所有问题。"},
]

while True:
    question = input("👤你：")
    if question == "退出":
        print("👋 再见！")
        break

    messages.append({"role": "user", "content": question})
    try:
        response = client.chat.completions.create(
            model=model,
            messages=messages,
            temperature=0.7,
        )
        reply_content = response.choices[0].message.content
        messages.append({"role": "assistant", "content": reply_content})
        print(f"🤖 助手：{reply_content}")
    except Exception as e:
        print(f"发生错误：{e}")
```

### 3.2 step3_api.py（最终版）

```python
import os
from dotenv import load_dotenv
from openai import OpenAI
from fastapi import FastAPI
from fastapi.responses import HTMLResponse
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
import uvicorn

load_dotenv()
api_key = os.getenv("DEEPSEEK_API_KEY")
base_url = os.getenv("DEEPSEEK_BASE_URL")
model = "deepseek-chat"
client = OpenAI(api_key=api_key, base_url=base_url)

app = FastAPI(title="个人助理 API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)

class ChatRequest(BaseModel):
    question: str

class ChatResponse(BaseModel):
    reply: str

messages = [
    {"role": "system", "content": "你是一个乐于助人的个人助理，用中文回答所有问题。"},
]

@app.post("/chat", response_model=ChatResponse)
def chat(req: ChatRequest):
    messages.append({"role": "user", "content": req.question})
    response = client.chat.completions.create(
        model=model,
        messages=messages,
        temperature=0.7,
    )
    reply_content = response.choices[0].message.content
    messages.append({"role": "assistant", "content": reply_content})
    return ChatResponse(reply=reply_content)

@app.get("/", response_class=HTMLResponse)
def index():
    with open("static/index.html", "r", encoding="utf-8") as f:
        return f.read()

if __name__ == "__main__":
    uvicorn.run(app, host='0.0.0.0', port=1000)
```
