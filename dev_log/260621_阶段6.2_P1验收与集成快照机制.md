- **只备份 app.py**：agent/、前端、RAG 等用 Git 管，避免快照占磁盘

## 二、架构决策

### 决策 1：主入口命名

| 选项 | 结论 |
|------|------|
| 继续 `step23_rag.py` | ❌ 阶段命名过时 |
| 每次阶段改文件名 | ❌ import 链、文档、启动命令全要改 |
| **`app.py` 稳定名 + 快照存历史** | ✅ 采用 |

### 决策 2：P1.1 记忆路由

用户原话：「用户正常使用不会故意显露偏好/画像/记忆类问题，优化完全没必要。」

结论：**不做**关键词硬路由。后续若需优化，应从 `result_synthesis` prompt 或路由器语义理解入手，而非规则匹配。

### 决策 3：快照 vs Git

- Git：版本控制、diff、协作
- 快照：集成里程碑的 **app.py 副本**，恢复直观、不依赖 git 知识
- 两者互补，快照不替代 git

### 决策 4：快照备份范围

| 选项 | 结论 |
|------|------|
| 默认备份 15 个文件（agent/、前端、MEMORY 等） | ❌ 占磁盘，不符合「里程碑入口」初衷 |
| **只存 `app.py`** | ✅ 采用；去掉 `-f` 额外文件参数 |
