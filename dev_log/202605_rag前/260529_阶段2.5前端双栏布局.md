## 二、前端 JS 核心认知（脑雾友好版）

以下内容不要求会写，理解"前端在做什么"即可。以后让 AI 写前端时，描述这三样就够了。

### 前端 JS 只做一件事

```
用户操作 → JS 捕获事件 → 调后端 API → 拿到数据 → 更新页面
```

所有函数都是这四步的变体，没有例外。

### 四个核心概念

#### 概念 1：找到页面元素（getElementById）

HTML 里写了 `<div id="messages">`，JS 通过 `id` 找到它，存到变量里，之后就能操作它了（改文字、加子元素、删掉）。

类比 Python：
```python
messagesDiv = find_widget_by_id("messages")  # 拿到引用
messagesDiv.text = "你好"                     # 改文字
```

本页面 JS 需要操作的 5 个元素：

| 元素 id | 作用 | JS 怎么用 |
|---------|------|-----------|
| `messages` | 消息显示区 | 往里加气泡、清空、滚到底部 |
| `question` | 输入框 | 读用户打字 `.value`、清空、聚焦 |
| `sendBtn` | 发送按钮 | 禁用/恢复 `.disabled`、改文字 |
| `sessionList` | 侧边栏列表 | 往里加会话条目、清空重建 |
| `chatHeader` | 聊天区标题 | 切换会话时更新文字 |

#### 概念 2：事件驱动

JS 代码不是从头执行到尾就结束了。它是"等着"——等用户点击、等键盘按下、等页面加载完，然后执行对应的函数。

这是前端和后端最大的思维差异：
- **后端**：请求来了 → 处理 → 返回响应 → 结束
- **前端**：页面一直开着，随时等用户操作，随时响应

```js
window.onload = init;                          // 页面加载完 → 执行 init()
deleteBtn.onclick = function() { ... };        // 点击删除 → 执行删除逻辑
questionInput.addEventListener("keydown", ...) // 按键 → 检查是不是 Enter
```

#### 概念 3：异步 + fetch（调后端 API）

```js
const response = await fetch("/sessions");   // 发 HTTP GET 请求
const data = await response.json();           // 解析返回的 JSON
```

对应 Python：
```python
response = requests.get("http://localhost:1000/sessions")
data = response.json()
```

JS 里多出来的 `await` 是因为浏览器是单线程的，同步等网络会把整个页面卡死。`await` 让浏览器在等网络时还能响应用户。

#### 概念 4：动态创建 DOM

因为页面上有多少会话、多少消息是**运行时才知道**的（后端返回的数据），所以不能在 HTML 里写死。必须用 JS 动态创建元素：

```js
const div = document.createElement("div");   // 创建一个 <div> 元素（还没加到页面上）
div.className = "session-item";              // 相当于 <div class="session-item">
div.textContent = "你好";                     // 相当于 <div>你好</div>
parent.appendChild(div);                     // 把这个 div 插到父元素里，真正显示出来
```

类比 Python（但不完全一样）：
```python
# JS 是创建对象节点，不是拼字符串。对象节点方式更安全（防 XSS 注入攻击）。
div = {"tag": "div", "class": "session-item", "text": "你好"}
parent.children.append(div)
```

## 三、页面生命周期（完整数据流）

```
页面打开
  │
  └→ window.onload 触发 init()
       │
       ├─ GET /sessions → 拿到会话列表
       │
       ├─ 有历史会话？
       │   ├─ 是 → renderSessionList(画侧边栏) → switchSession(加载消息)
       │   └─ 否 → createSession(新建空会话)
       │
       └─ 页面渲染完毕，等待用户操作...

用户操作          →  前端函数           →  后端 API                         →  页面变化
──────────────────────────────────────────────────────────────────────────────────
点侧边栏会话       →  switchSession()   →  GET /sessions/{id}/messages      →  聊天区显示消息
点 + 号           →  createSession()   →  POST /sessions                   →  清空聊天区，显示"新会话"
点 × 删除         →  deleteSession()   →  DELETE /sessions/{id}            →  刷新侧边栏，切到其他会话
输入文字按 Enter  →  sendMessage()     →  POST /chat {session_id, question} →  显示用户气泡 + AI 回复
```

所有函数都在做同一件事：**调 API → 拿数据 → 更新页面上的 DOM**。

```html
        /**
         * init() — 页面初始化入口
         * 执行流程：
         *   1. 调 GET /sessions 获取已有会话列表
         *   2. 如果有会话 → 切换到最近一个
         *   3. 如果没有 → 自动创建一个新会话
         */
        async function init() {
            // ---- 3.1 获取服务器上已有的会话列表 ----
            // fetch() 是浏览器内置的 HTTP 客户端，相当于 Python 的 requests.get()，发请求
            // await 等待网络请求完成（不会卡住页面），拿到响应后再继续
            const response = await fetch("/sessions");

            // response.json() 把 HTTP 响应体（JSON 字符串）解析成 JS 数组
            // 后端返回格式：[{session_id, title, message_count, updated_at}, ...]
            const sessions = await response.json();

            if (sessions.length > 0) {
                // ---- 3.2 有历史会话：切换到最近一个 ----
                // 后端已按 updated_at DESC 排序，所以 sessions[0] 就是最近更新的
                // 先渲染侧边栏列表，再切换到第一个
                renderSessionList(sessions);
                await switchSession(sessions[0].session_id);
            } else {
                // ---- 3.3 没有历史会话：自动创建一个 ----
                // 新建会话不需要传参数，后端生成 UUID 返回
                await createSession();
            }
        }

        // ============================================================
        // 4. loadSessionList() — 从后端拉取会话列表
        // ============================================================
        /**
         * 调 GET /sessions 获取最新会话列表，并更新侧边栏
         * 这个函数在 创建/删除/发消息 后都会调用，保持侧边栏和服务器同步
         *
         * @returns {Array} 会话数组，供调用者进一步使用
         */
        async function loadSessionList() {
            const response = await fetch("/sessions");
            const sessions = await response.json();
            // renderSessionList 只负责画 UI，数据还是 sessions
            renderSessionList(sessions);
            return sessions;
        }

        // ============================================================
        // 5. renderSessionList() — 把会话数组渲染成侧边栏的 DOM 元素
        // ============================================================
        /**
         * DOM = Document Object Model（文档对象模型）
         * 浏览器把 HTML 解析成一棵树，JS 可以通过 DOM API 操作这棵树（增删改查元素）
         *
         * 这个函数做的事情：
         *   1. 清空侧边栏旧内容
         *   2. 遍历 sessions 数组
         *   3. 每个 session 创建一个 <div class="session-item"> 元素
         *   4. 插到侧边栏里
         *
         * @param {Array} sessions — 会话对象数组 [{session_id, title, message_count, updated_at}, ...]
         */
        function renderSessionList(sessions) {
            // ---- 5.1 清空旧列表 ----
            // innerHTML = "" 相当于把 div 里的内容全部删掉
            sessionListDiv.innerHTML = "";

            // ---- 5.2 遍历数组，逐条创建 DOM 元素 ----
            // forEach 相当于 Python: for session in sessions:
            sessions.forEach(function(session) {
                // ---- 5.2.1 创建会话条目的外层 div ----
                const itemDiv = document.createElement("div");
                // className 设置 CSS class，决定样式
                itemDiv.className = "session-item";
                // 如果这个会话是当前选中的，加上 active 类名（高亮显示）
                if (session.session_id === currentSessionId) {
                    itemDiv.className += " active";
                }

                // ---- 5.2.2 创建文字信息区（标题 + 元信息） ----
                const infoDiv = document.createElement("div");
                infoDiv.className = "session-info";

                // 标题行：用户第一条消息的前几个字，或 "新会话"
                const titleDiv = document.createElement("div");
                titleDiv.className = "session-title";
                // textContent 设置元素的文字内容，和 Python 的 label["text"] = "xxx" 一样
                titleDiv.textContent = session.title || "新会话";

                // 元信息行："5 条消息 · 05-28 14:30"
                const metaDiv = document.createElement("div");
                metaDiv.className = "session-meta";
                // formatTime() 是我们自己写的辅助函数，把 ISO 时间戳转成易读格式
                metaDiv.textContent = session.message_count + " 条消息 · " + formatTime(session.updated_at);

                // appendChild 把子元素加到父元素里
                // 结构：infoDiv 包含 titleDiv 和 metaDiv
                infoDiv.appendChild(titleDiv);
                infoDiv.appendChild(metaDiv);

                // ---- 5.2.3 创建删除按钮 ----
                const deleteBtn = document.createElement("button");
                deleteBtn.className = "btn-delete-session";
                deleteBtn.textContent = "×";  // × 是乘号，视觉上像关闭按钮

                // onclick 绑定点击事件：点击时执行箭头函数
                // event.stopPropagation() 阻止事件冒泡——点击删除按钮不应该触发切换会话
                // 如果不去掉这行，点删除会同时触发 deleteSession() 和 switchSession()
                deleteBtn.onclick = function(event) {
                    event.stopPropagation();   // 阻止事件冒泡到父元素 .session-item
                    deleteSession(session.session_id);
                };

                // ---- 5.2.4 组装：把子元素加到父 div 里 ----
                itemDiv.appendChild(infoDiv);
                itemDiv.appendChild(deleteBtn);

                // ---- 5.2.5 点击会话条目 → 切换到该会话 ----
                itemDiv.onclick = function() {
                    switchSession(session.session_id);
                };

                // ---- 5.2.6 把组装好的条目加到侧边栏 ----
                sessionListDiv.appendChild(itemDiv);
            });
        }

        // ============================================================
        // 6. switchSession() — 切换到指定会话，加载并显示其消息
        // ============================================================
        /**
         * 切换当前活跃会话的核心函数。
         * 做的事情：
         *   1. 更新全局 currentSessionId
         *   2. 从后端拉取该会话的所有消息（GET /sessions/{id}/messages）
         *   3. 渲染消息到聊天区
         *   4. 刷新侧边栏高亮
         *
         * @param {string} sessionId — 要切换到的会话 ID
         */
        async function switchSession(sessionId) {
            // ---- 6.1 更新当前会话 ID ----
            currentSessionId = sessionId;

            // ---- 6.2 从后端加载该会话的消息 ----
            // 后端返回格式：[{role, content, created_at}, ...]
            const response = await fetch("/sessions/" + sessionId + "/messages");
            const messages = await response.json();

            // ---- 6.3 渲染消息 ----
            renderMessages(messages);

            // ---- 6.4 刷新侧边栏（更新高亮状态） ----
            // 先拉最新列表再渲染，确保标题等信息也是最新的
            await loadSessionList();

            // ---- 6.5 更新聊天区标题 ----
            // 从侧边栏找当前会话的标题（loadSessionList 已经更新了 DOM）
            updateChatHeader();
        }

        /**
         * updateChatHeader() — 根据当前会话更新聊天区顶部标题
         * 从侧边栏 DOM 中找到当前活跃的 .session-item，读取其标题文字
         */
        function updateChatHeader() {
            // querySelector 用 CSS 选择器查找元素，返回第一个匹配的元素
            // ".session-item.active .session-title" 表示：class 同时有 session-item 和 active 的元素内部的 .session-title
            const activeItem = document.querySelector(".session-item.active .session-title");
            if (activeItem) {
                chatHeader.textContent = activeItem.textContent;
            } else {
                chatHeader.textContent = "个人 AI 助理";
            }
        }

        // ============================================================
        // 7. renderMessages() — 把消息数组渲染成聊天气泡
        // ============================================================
        /**
         * 清空消息区，然后根据 messages 数组逐条创建气泡。
         * 如果消息数组为空，显示"空状态"提示。
         *
         * @param {Array} messages — [{role, content, created_at}, ...]
         */
        function renderMessages(messages) {
            // ---- 7.1 清空消息区 ----
            messagesDiv.innerHTML = "";

            // ---- 7.2 处理空消息列表 ----
            if (messages.length === 0) {
                // 创建一个空的提示 div
                const emptyDiv = document.createElement("div");
                emptyDiv.className = "empty-state";
                emptyDiv.textContent = "开始新对话吧！";
                messagesDiv.appendChild(emptyDiv);
                return;
            }

            // ---- 7.3 逐条创建消息气泡 ----
            // forEach 遍历数组，对每条消息调用 addMessageBubble
            messages.forEach(function(msg) {
                // 跳过 system 消息（system prompt 是给 AI 的指令，用户不需要看到）
                if (msg.role === "system") return;

                // addMessageBubble 创建一条消息的 DOM 结构并加到页面上
                addMessageBubble(msg.role, msg.content, msg.created_at);
            });
        }

        /**
         * addMessageBubble() — 在消息区末尾追加一条消息气泡
         *
         * 每条消息的 HTML 结构：
         * <div class="message-wrapper user|assistant">
         *     <div class="message-bubble">消息内容</div>
         *     <div class="message-time">14:30</div>
         * </div>
         *
         * @param {string} role  — "user" 或 "assistant"，决定气泡颜色和位置
         * @param {string} text  — 消息文字内容
         * @param {string} isoTime — ISO 格式时间戳 "2026-05-28T14:30:00"
         */
        function addMessageBubble(role, text, isoTime) {
            // ---- 创建外层包裹 div ----
            const wrapper = document.createElement("div");
            wrapper.className = "message-wrapper " + role;  // "message-wrapper user" 或 "message-wrapper assistant"

            // ---- 创建气泡 div ----
            const bubble = document.createElement("div");
            bubble.className = "message-bubble";
            bubble.textContent = text;

            // ---- 创建时间戳 div ----
            const timeDiv = document.createElement("div");
            timeDiv.className = "message-time";
            // formatTime() 把 ISO 时间转成 "HH:MM" 短格式
            timeDiv.textContent = isoTime ? formatTime(isoTime) : "";

            // ---- 组装 ----
            wrapper.appendChild(bubble);
            wrapper.appendChild(timeDiv);
            messagesDiv.appendChild(wrapper);

            // ---- 自动滚到底部 ----
            // scrollTop 是当前滚动到的位置，scrollHeight 是整个可滚动区域的高度
            // 把 scrollTop 设为 scrollHeight 就滚到了最底部
            messagesDiv.scrollTop = messagesDiv.scrollHeight;
        }

        // ============================================================
        // 8. createSession() — 创建新会话
        // ============================================================
        /**
         * 调 POST /sessions 创建新会话，然后切换到空聊天界面。
         * 注意：空会话在数据库中没有任何消息记录，
         * 所以 GET /sessions 不会返回它。只有发了第一条消息后，
         * save_message() 写入数据库，这个会话才会出现在侧边栏。
         */
        async function createSession() {
            // ---- 8.1 调后端创建新会话 ----
            const response = await fetch("/sessions", { method: "POST" });
            const data = await response.json();        // {session_id: "abc123..."}

            // ---- 8.2 切换到这个空会话 ----
            currentSessionId = data.session_id;

            // ---- 8.3 清空消息区，显示空状态 ----
            messagesDiv.innerHTML = "";
            const emptyDiv = document.createElement("div");
            emptyDiv.className = "empty-state";
            emptyDiv.textContent = "新会话，开始聊天吧！";
            messagesDiv.appendChild(emptyDiv);

            // ---- 8.4 刷新侧边栏 + 更新标题 ----
            await loadSessionList();
            chatHeader.textContent = "新会话";
            questionInput.focus();  // 让输入框获得焦点，用户可以直接打字
        }

        // ============================================================
        // 9. deleteSession() — 删除指定会话
        // ============================================================
        /**
         * 调 DELETE /sessions/{id} 删除会话。
         * 删除后：
         *   - 如果删的是当前会话 → 自动切换到列表中的第一个会话
         *   - 如果列表也空了 → 自动创建新会话
         *
         * @param {string} sessionId — 要删除的会话 ID
         */
        async function deleteSession(sessionId) {
            // ---- 9.1 确认对话框 ----
            // confirm() 是浏览器内置函数，弹出确认/取消对话框
            // 用户点"取消"返回 false，直接 return 不删
            if (!confirm("确定要删除这个会话吗？此操作不可恢复。")) return;

            // ---- 9.2 调后端删除 ----
            await fetch("/sessions/" + sessionId, { method: "DELETE" });

            // ---- 9.3 重新加载会话列表 ----
            const sessions = await loadSessionList();

            // ---- 9.4 如果删的是当前会话，需要切换 ----
            if (sessionId === currentSessionId) {
                if (sessions.length > 0) {
                    // 还有其他会话 → 切换到第一个
                    await switchSession(sessions[0].session_id);
                } else {
                    // 没有会话了 → 自动创建一个
                    await createSession();
                }
            }
        }

        // ============================================================
        // 10. sendMessage() — 核心函数：发消息 → 调 API → 显示回复
        // ============================================================
        /**
         * 执行流程：
         *   1. 读取输入框内容
         *   2. 如果还没有会话 → 自动创建一个
         *   3. 显示用户气泡
         *   4. 调 POST /chat（带上 session_id 和 question）
         *   5. 显示助手气泡
         *   6. 刷新侧边栏（标题和消息数可能变了）
         */
        async function sendMessage() {
            // ---- 10.1 获取用户输入 ----
            const question = questionInput.value.trim();
            if (question === "") return;  // 空内容不发送

            // ---- 10.2 如果还没有会话，自动创建 ----
            // 这处理了"页面刚打开，用户直接打字"的场景
            if (!currentSessionId) {
                await createSession();
            }

            // ---- 10.3 在页面上显示用户消息 ----
            // 先清空空状态提示（如果有的话），再添加气泡
            clearEmptyState();
            addMessageBubble("user", question, new Date().toISOString());

            // ---- 10.4 清空输入框 + 禁用按钮 ----
            questionInput.value = "";
            sendBtn.disabled = true;
            sendBtn.textContent = "思考中...";

            try {
                // ---- 10.5 调用后端 POST /chat ----
                // 后端 ChatRequest 模型要求两个字段：session_id 和 question
                const response = await fetch("/chat", {
                    method: "POST",
                    headers: { "Content-Type": "application/json" },
                    // JSON.stringify 把 JS 对象转成 JSON 字符串
                    body: JSON.stringify({
                        session_id: currentSessionId,
                        question: question,
                    }),
                });

                const data = await response.json();  // {created_at: "...", reply: "..."}

                // ---- 10.6 显示助手回复 ----
                addMessageBubble("assistant", data.reply, data.created_at);

                // ---- 10.7 刷新侧边栏 ----
                // 第一次发消息后，该会话才出现在数据库里，所以要刷新列表
                await loadSessionList();
                // 更新标题（第一次发消息后标题从"新会话"变成用户第一句话）
                updateChatHeader();

            } catch (error) {
                // 网络挂了或服务器崩了，显示错误提示
                addMessageBubble("assistant", "网络错误：" + error.message, null);
            }

            // ---- 10.8 恢复按钮 ----
            sendBtn.disabled = false;
            sendBtn.textContent = "发送";
            questionInput.focus();
        }

        /**
         * clearEmptyState() — 清除消息区的空状态提示
         * 在发第一条消息时调用，把"开始新对话吧"换成实际消息
         */
        function clearEmptyState() {
            // querySelector 返回第一个匹配的元素，没有则返回 null
            const emptyEl = messagesDiv.querySelector(".empty-state");
            if (emptyEl) {
                // remove() 把元素从 DOM 树中删掉
                emptyEl.remove();
            }
        }

        // ============================================================
        // 11. 工具函数 — formatTime()
        // ============================================================
        /**
         * 把 ISO 格式时间戳转成易读的中文格式。
         * 规则：
         *   - 今天的消息 → 只显示 "HH:MM"（如 "14:30"）
         *   - 今年的消息 → 显示 "MM-DD HH:MM"（如 "05-28 14:30"）
         *   - 更早的消息 → 显示完整 "YYYY-MM-DD HH:MM"
         *
         * @param {string} isoStr — ISO 格式时间戳，如 "2026-05-28T14:30:00"
         * @returns {string} 格式化后的时间字符串
         */
        function formatTime(isoStr) {
            if (!isoStr) return "";  // 防御：没有时间戳就返回空字符串

            // new Date(isoStr) 把 ISO 字符串解析为 JS 的 Date 对象
            // Date 对象类似于 Python 的 datetime 对象
            const date = new Date(isoStr);
            const now = new Date();

            // getHours() 返回小时(0-23)，getMinutes() 返回分钟(0-59)
            // toString().padStart(2, "0") 确保两位数：5 → "05"
            const time = date.getHours().toString().padStart(2, "0") + ":" +
                         date.getMinutes().toString().padStart(2, "0");

            // getMonth() 返回 0-11，所以 +1
            const month = (date.getMonth() + 1).toString().padStart(2, "0");
            const day = date.getDate().toString().padStart(2, "0");

            // toDateString() 返回 "Thu May 28 2026" 格式，用于比较是不是同一天
            if (date.toDateString() === now.toDateString()) {
                // 今天的消息，只显示时间
                return time;
            }

            // getFullYear() 返回四位年份
            if (date.getFullYear() === now.getFullYear()) {
                // 今年的消息，显示月-日 时间
                return month + "-" + day + " " + time;
            }

            // 更早的消息，显示完整日期
            return date.getFullYear() + "-" + month + "-" + day + " " + time;
        }

        // ============================================================
        // 12. 键盘事件 — 按 Enter 发送消息
        // ============================================================
        // addEventListener("keydown", fn) 给元素绑定"按键被按下"事件
        // 当用户在输入框按下任意键时，浏览器调用我们提供的匿名函数
        questionInput.addEventListener("keydown", function(event) {
            // event.key 是按下的是哪个键，如 "Enter"、"Escape"
            // !event.shiftKey：按住 Shift 再按 Enter 不触发发送（方便用户换行）
            if (event.key === "Enter" && !event.shiftKey) {
                // preventDefault() 阻止浏览器对这个按键的默认行为
                // 输入框里按 Enter 默认可能会触发表单提交
                event.preventDefault();
                sendMessage();
            }
        });
    </script>

</body>
</html>
```

## 六、关键设计决策

### 1. 空会话不出现在侧边栏

`list_sessions()` 的数据源是 `messages` 表。刚创建但没发消息的会话在表里没有记录，查不出来。

**流程**：POST /sessions → 前端内存持有 session_id → 用户发第一条消息 → save_message 写入 DB → GET /sessions 才能查到 → 侧边栏才出现该会话。

### 2. 事件冒泡与 stopPropagation

删除按钮在会话条目**内部**。会话条目绑了 `onclick` 切换会话，删除按钮绑了 `onclick` 删除会话。如果不阻止冒泡，点删除按钮会同时触发两个事件——先删了会话，又尝试切到这个已删除的会话。

`event.stopPropagation()` 阻止事件从删除按钮向上"冒泡"到会话条目，确保只执行删除逻辑。

### 3. 删除当前会话后的处理

```js
if (sessionId === currentSessionId) {
    if (sessions.length > 0) {
        await switchSession(sessions[0].session_id);  // 切到第一个
    } else {
        await createSession();  // 全删光了，自动建新的
    }
}
```

### 4. HTML 结构变化对照

| 旧结构（单栏） | 新结构（双栏） |
|---|---|
| `<div class="chat-container">` 居中 600px | `<div class="app-layout">` flex 横向全屏 |
| 无侧边栏 | `<div class="sidebar">` 280px 深色背景 |
| 消息区 + 输入区在 chat-container 内 | `<div class="chat-area">` 包含头部 + 消息区 + 输入区 |
| 消息气泡直接是 `.message` div | 消息包裹 `.message-wrapper` > 气泡 + 时间戳 |

### 5. 前端 ↔ 后端字段对齐

| 方向 | API | 前端发送 | 后端返回 |
|------|-----|---------|---------|
| 前端→后端 | POST /chat | `{session_id, question}` | `{created_at, reply}` |
| 前端→后端 | POST /sessions | （空） | `{session_id}` |
| 前端←后端 | GET /sessions | — | `[{session_id, title, message_count, updated_at}]` |
| 前端←后端 | GET /sessions/{id}/messages | — | `[{role, content, created_at}]` |
| 前端→后端 | DELETE /sessions/{id} | — | `{ok: true}` |
