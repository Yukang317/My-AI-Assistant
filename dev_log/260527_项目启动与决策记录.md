**日期**：2026-05-27

## 1. 项目背景

用户（Yukang317）希望将原有的 SAGT 销冠智能体项目重构为通用个人 AI 助理，涵盖工作、生活和学习多方面。

### 原有基础
- SAGT 项目：LangGraph + FastAPI + 企业微信 + PostgreSQL/Redis
- 项目是跟着老师做的，用户对整体有理解但独立编码能力较弱
- 用户有重度抑郁焦虑症状，影响认知功能

### 核心诉求
- 个人助理能根据用户实际情况回复合适内容
- 需要 RAG 能力处理大量个人资料
- 需要完善的记忆系统
- 能接入企业微信/飞书等平台随时随地聊天
- 目标是达到企业认可、能拿高薪 offer 的水平

---

## 2. 技术决策

### 协作方式
采用"施工图纸"模式，分三步走：
1. Claude 给出施工图纸（要建什么文件、什么函数、输入输出）
2. 用户手写代码
3. Claude review + 补充

### 分阶段路线

| 阶段 | 目标 | 说明 |
|------|------|------|
| 阶段1 | 最简单 Web 聊天 | FastAPI + 单 HTML + DeepSeek API + 内存存储 |
| 阶段2 | 持久记忆 | 接入 SQLite，对话历史持久化 |
| 阶段3 | 知识库 RAG | 个人文档向量化 + 检索增强生成 |
| 阶段4 | 多平台接入 | 飞书/企微 + 高级功能 |

### 阶段1 技术选型
- LLM：DeepSeek API（OpenAI 兼容格式）
- 后端：FastAPI（用户熟悉）
- 前端：单文件 HTML（不引入框架）
- 存储：内存 dict（阶段2 换 SQLite）
- 运行：`uv run` 直接启动

### 刻意不用的技术（现阶段）
- LangGraph（简单请求-响应不需要图编排）
- PostgreSQL/Redis（太重）
- 企业微信 SDK（先做 Web）
- 复杂 Pydantic 模型（简单 dict 就行）
