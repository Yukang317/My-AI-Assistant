## 关键概念

### 1. 空会话不会出现在侧边栏

`list_sessions()` 的 SQL 查询数据源是 `FROM messages m`。刚创建但没发消息的会话在 `messages` 表里没有记录，查不出来。

流程：POST /sessions → 前端内存持有 session_id → 用户发第一条消息 → save_message 写入 DB → GET /sessions 才能查到。

### 2. 路径参数

FastAPI 中 `@app.delete("/sessions/{session_id}")` 的 `{session_id}` 会自动从 URL 提取值传给同名函数参数。这是 FastAPI 内置特性，不需要额外装饰器。

### 3. RESTful 资源设计

| 动作 | HTTP 方法 | URL |
|------|-----------|-----|
| 新建 | POST | `/sessions` |
| 列表 | GET | `/sessions` |
| 删除 | DELETE | `/sessions/{id}` |
| 查看 | GET | `/sessions/{id}/messages` |

### 4. 前端需要的 HTTP 操作

除了已有的 `POST /chat`，前端还需要：
- `GET /sessions` — 加载侧边栏列表
- `POST /sessions` — 新建会话按钮
- `DELETE /sessions/{id}` — 删除会话按钮
- `GET /sessions/{id}/messages` — 点击会话时加载消息

### 5. 前端事件流思维模型

```
用户操作           →  前端 JS                  →  后端端点
──────────────────────────────────────────────────────────
页面加载            →  GET /sessions            →  返回会话列表 → 渲染侧边栏
点"新建会话"        →  POST /sessions           →  返回新 session_id → 设为当前 + 清空聊天区
点某个会话          →  GET /sessions/{id}/messages → 返回消息列表 → 渲染聊天区
点删除按钮          →  DELETE /sessions/{id}    →  删库 → 刷列表 + 切走
发消息（已有）       →  POST /chat              →  调 LLM → 存储 → 返回回复
```
