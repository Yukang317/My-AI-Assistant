# 开发工具 — Dev-Logger Skill 创建

**日期**：2026-06-02
**内容**：为 personal_assistant 项目创建一个自动记录开发日志的 Claude Code Skill，用于规范开发日志的编写和项目状态文件的更新

---

## 〇、做了什么 & 为什么

### 背景

之前的开发日志（`dev_log/` 目录下的 9 个文件）和项目状态文件（`上次会话成果.md`、`准备工作总结.md`）都是手动维护的。每次会话结束后需要：
1. 回忆会话中做了什么
2. 按格式写开发日志
3. 更新多个状态文件

这个流程容易遗漏，而且格式不统一。为了解决这个问题，决定创建一个 Claude Code Skill 来自动化这个过程。

### Skill 的作用

把"记录开发日志"这件事从"靠人记住要做"变成"系统自动提醒+执行"，确保每次开发工作都有完整的记录。

---

## 一、Skill 设计核心

### 1.1 触发机制（双重保障）

| 触发方式 | 条件 | 说明 |
|---------|------|------|
| **主动触发** | 用户说"记录一下""写日志""收尾"等关键词 | 立即执行 |
| **自动提醒** | 会话中有代码产出/技术讨论/工作告一段落 | 主动询问是否需要记录 |

### 1.2 日志格式（参照 RAG 项目实战风格）

日志模板包含 7 个部分：
1. **做了什么 & 为什么** — 本次工作在项目中的位置和意义
2. **代码全貌** — 关键代码片段（不是逐行拆解，聚焦新增部分）
3. **自提疑问 & 解答** — Q&A 形式，记录讨论中涉及的疑问和解答
4. **踩坑记录** — bug 表格（现象→根因→修复）
5. **协作模式评估** — A/B/C/D 模式使用效果
6. **文件状态** — 本次涉及文件的完成状态
7. **下一步** — 接下来做什么

### 1.3 需要更新的四个文件

```
上次会话成果.md     ← 每次必更新（最重要）
准备工作总结.md     ← 有实质变化时更新（文件结构/技术决策/进度）
CLAUDE.md          ← 有架构变化时更新（目录/命令/模块）
项目需求文档.md     ← 有需求变化时更新（路线图/选型）
```

**设计原则**：只更新确实有变化的内容，不做无意义的同步。

### 1.4 写作风格

参照 `学习过程中的RAG项目实战/开发日志笔记.md`：
- 对话式语气（"你"而不是"用户"）
- 问题驱动（以"遇到的疑问"为线索组织内容）
- 自提疑问是核心（最有价值的部分）
- 类比解释技术概念
- Bug 记录要完整（现象→原因→修复）

但粒度控制在**核心讨论+代码+Bug**级别，不需要逐行拆解已知代码。

## 二、自提疑问 & 解答

### Q1：为什么 Skill 放在 `.claude/skills/` 而不是 `personal_assistant/` 下？

**解答**：`.claude/skills/` 是 Claude Code 的标准 Skill 目录，放在这里：
- Claude Code 会自动发现和加载
- 不需要修改任何配置文件
- Skill 是给 Claude 用的"操作手册"，不是项目源码的一部分

这类似于 `.claude/settings.json` 和 `.claude/CLAUDE.md`——它们是开发环境配置，不属于项目交付物。

### Q2：日志更新是"自动"还是"手动确认后"执行？

**解答**：采用**半自动**模式——自动提醒 + 用户确认后执行。

**为什么不完全自动？**
- 日志质量比数量重要，写错了反而增加维护负担
- 用户可能觉得某次工作不值得记录（比如只是看代码、没改东西）
- 写日志本身需要用户确认内容是否准确——毕竟 Claude 不可能 100% 准确回忆起整个会话

**流程**：
```
Claude 主动提醒"需要记录吗？"
  → 用户确认"记录一下"
    → Claude 写日志 + 更新状态文件
    → 展示给用户确认
```

### Q3：四个状态文件的分工是什么？会不会重复？

**解答**：各有分工，不会重复：

| 文件 | 谁读 | 存什么 | 更新频率 |
|------|------|--------|---------|
| `dev_log/xxx.md` | 你自己（回顾学习过程） | 详细的讨论、代码、Bug、疑问 | 每次会话 |
| `上次会话成果.md` | Claude（新会话加载） | 上一步做了什么、下一步做什么 | 每次会话 |
| `准备工作总结.md` | Claude（新会话加载） | 项目常量：用户档案、技术决策、文件结构 | 偶尔 |
| `CLAUDE.md` | Claude（每次对话） | 仓库级别的指令和架构说明 | 偶尔 |

它们的读者不完全相同，粒度也不同。`dev_log` 是给人看的（详细），状态文件是给 Claude 看的（摘要）。
