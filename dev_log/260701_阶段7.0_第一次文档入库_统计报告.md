### 模型选择说明

最初配置为 `BAAI/bge-large-zh-v1.5`，该模型最大输入仅 **512 tokens**（约 500 中文字符），而 parent chunk 设置为 2048 字符，远超限制，导致全部 66 个文件 API 调用返回 400。切换至 `BAAI/bge-m3` 后问题解决：

- **bge-m3**：8192 tokens 最大输入，多语言支持，1024 维
- 另一已修复问题：OpenAI SDK v2.38 默认 `encoding_format='base64'`，硅基流动不支持，已在 `rag/embedding.py` 显式传 `"float"`

## 问题与修复

### Bug 1：全部文件 400 错误

| 维度 | 详情 |
|------|------|
| **现象** | 66 文件全部 `openai.BadRequestError: 400 - 'The parameter is invalid.'` |
| **根因 1** | OpenAI SDK v2.38 默认 `encoding_format='base64'`，硅基流动不支持 |
| **修复 1** | `rag/embedding.py` L52 显式传 `encoding_format="float"` |
| **根因 2** | `BAAI/bge-large-zh-v1.5` 最大输入 512 tokens，parent chunk 2048 字符远超限制 |
| **修复 2** | `.env` 中设置 `EMBEDDING_API_MODEL=BAAI/bge-m3`（8192 tokens 上限） |

### Bug 2：`.pyc` 缓存导致修改不生效

修改 `embedding.py` 后测试仍失败，因为 `__pycache__/` 中的旧 `.pyc` 未被覆盖。
执行 `find . -type d -name "__pycache__" -exec rm -rf {} +` 解决。
