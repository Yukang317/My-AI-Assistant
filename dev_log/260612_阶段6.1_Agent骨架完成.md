## 二、Agent 最终架构

```
START
  │
  ▼
load_context        ← PG 历史 + MEMORY.md
  │
  ▼
intent_route        ← LLM 意图识别（JSON prefill）
  │
  ├─ general_chat ───────────────┐
  │                              │
  └─ use_tool                     │
       │                          │
       ▼                          │
  tool_execute     ← 工具注册表   │
       │                          │
       └──────────────────────────┘
                    │
                    ▼
            result_synthesis   ← LLM 合成 + PG 保存
                    │
                    ▼
                   END
```

### 关键设计点

| 设计 | 说明 |
|------|------|
| 薄封装适配器 | `intent_node`/`result_synthesis_node` 闭包桥接 `(state) -> dict` 和 `(state, model) -> dict` |
| MemorySaver 检查点 | `thread_id = session_id`，每次节点执行后自动存档，下次调用从存档恢复 |
| JSON prefill | `AIMessage(content="{")` 引导 LLM 直接输出 JSON，从源头杜绝 markdown 噪声 |
| 工具结果放 HumanMessage | 防止 prompt injection（外部内容不进 SystemMessage） |
| 占位符模板 | `.format()` 一次性注入 memory_context + bound_summary，避免字符串拼接混乱 |

## 四、核心知识点

### LangGraph 节点签名约定

LangGraph 每个节点必须是 `(state: TypedDict) -> dict`。需要额外参数的函数必须用薄封装适配：

```python
# 业务函数需要 model 参数
def result_synthesis(state, model):
    ...

# 薄封装：闭包捕获 model
def result_synthesis_node(state):
    return result_synthesis(state, get_llm())
```

### MemorySaver 检查点机制

- `graph.compile(checkpointer=MemorySaver())` — 每次节点执行后自动保存 State
- 多轮对话靠 `thread_id` 隔离——同一 `thread_id` 从上次存档继续，不同 `thread_id` 互不干扰
- 内存存储（进程重启丢失），生产环境换 `SqliteSaver` 或 `PostgresSaver`

### JSON prefill 防 LLM 噪声

```python
messages = [
    SystemMessage(content="..."),
    HumanMessage(content=question),
    AIMessage(content="{"),  # LLM 看到 assistant 已输出 "{"，直接续写 JSON
]
```

LLM 不会再说 "好的，这是JSON：" 或包裹 ```json 标记——它认为自己已经在输出 JSON 了。

### 防 prompt injection

外部数据（工具搜索返回的内容、用户上传的文件）**绝不**放进 SystemMessage——SystemMessage 是"操作系统指令"角色，LLM 倾向无条件信任。同样，外部数据也不应通过 `+=` 拼接到 SystemMessage 中。正确方式：

```python
HumanMessage(content=f"{question}\n\n【工具检索结果】\n{results_text}")
```
