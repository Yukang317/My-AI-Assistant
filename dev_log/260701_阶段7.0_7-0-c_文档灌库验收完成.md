## 一、Bug 排查与修复

灌库脚本首次执行时，全部 66 个文件返回：
```
openai.BadRequestError: Error code: 400 - {'code': 20015, 'message': 'The parameter is invalid.'}
```

### Bug 1：`encoding_format` 不兼容

| 维度 | 详情 |
|------|------|
| **现象** | 硅基流动 API 返回 400 + `The parameter is invalid` |
| **根因** | OpenAI SDK v2.38 的 `embeddings.create()` 默认 `encoding_format='base64'`，硅基流动只接受 `float` |
| **定位方法** | 单文件短文本调 `embed()` 仍 400 → 排除文本长度 → 怀疑 SDK 默认参数 |
| **修复** | `rag/embedding.py` L52 显式传 `encoding_format="float"` |

### Bug 2：模型 token 上限与 chunk 大小不匹配（主因）

修复 Bug 1 后短文本成功，灌库仍大量 400：

| 测试文本长度 | 结果 |
|-------------|------|
| 500 字符 | ✅ 200 OK |
| 600 字符 | ❌ 400 |

**根因**：`BAAI/bge-large-zh-v1.5` 最大 **512 tokens**，但 `PARENT_CHUNK_SIZE=2048` 字符，实际 parent chunk 达 1655~1686 字符。

**为什么换模型而不是截断**：
- 截断丢失 chunk 后半语义，向量质量下降
- parent 2048 字是为 LLM 上下文设计的
- `BAAI/bge-m3` 支持 8192 tokens，维度同为 1024，**无需重建 Milvus Collection**

**修复**：`.env` 设置 `EMBEDDING_API_MODEL=BAAI/bge-m3`

## 五、知识点笔记

### Q1：bge-m3 和 bge-large-zh 维度都是 1024，是巧合吗？

不是。BGE 系列保持维度兼容，切换模型不必 `drop_collection` 重建 Milvus。

### Q2：为什么 parent/child 是 2 次 API 调用？

`embed()` 接收 `List[str]`，API 支持批量输入。每个文件 = 1 次 parent batch + 1 次 child batch，不是每个 chunk 一次。

### Q3：灌库脚本为什么不更新 BM25？

BM25 是内存索引，`app.py` 首次 RAG 请求时从 Milvus 全量 child_chunks 重建。离线脚本只写 Milvus + MinIO + PG，与上传 API 行为一致。

### Q4：filename 为什么用相对路径？

`DocumentIndexer._build_object_key()` 用 filename 拼 MinIO 路径。`dev_log/xxx.md` 比裸 `xxx.md` 能避免跨目录同名冲突，检索结果也能看出文档来源。
