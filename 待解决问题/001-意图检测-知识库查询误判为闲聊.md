# 001 — 意图检测对"知识库有X吗"类查询误判为闲聊

**状态：** 🔴 待讨论/未定案

## 问题描述

用户提问"知识库里有内容吗？"时，意图检测（intent detection）将其路由到了 `general_chat`（闲聊）路径，而非 `use_tool` → `rag_search` 路径。

这是典型的**隐式检索意图**问题——用户没有直接说"帮我搜索"，但语义上需要查询知识库才能回答。

## 复现步骤

在 `6-1test.py` 测试中，将测试2的问题改为 `"知识库里有内容吗？"`，观察 state 快照：
- 预期：`intent = use_tool`，`target = rag_search`
- 实际：触发了闲聊路径

## 当前行为 vs 期望行为

| | 当前 | 期望 |
|------|------|------|
| 意图 | `general_chat` | `use_tool` |
| 目标工具 | (无) | `rag_search` |
| 回复 | 闲聊式回复 | RAG 检索后生成回复 |

## 受影响的查询类型（推测）

以下类型的查询可能都存在类似问题：
- "知识库里有关于 X 的内容吗？"
- "你知道 X 吗？"
- "之前存过 X 的文档吗？"
- "有没有 X 的资料？"
- "能查到 X 吗？"

## 根本原因分析（⚠️ 推测，尚未验证）

意图检测节点（推测在 `agent/main_graph.py` 中的 intent detection 逻辑）对"检索隐含性"的识别不够——它可能过度依赖显式动作词（如"搜索""查一下""帮我找"），而把"有...吗""知道...吗"等疑问句归为闲聊。**此分析需要额外的测试和日志确认。**

## 可能的解决方向（⚠️ 均为待讨论选项，尚未评估可行性）

以下方向仅作为头脑风暴记录，**每一项都需要重新评估**后才能决定是否采用：

1. **Prompt 优化（待评估）**：在意图检测 prompt 中增加"关于知识库/资料的存在性询问 → use_tool"的规则和示例
2. **关键词辅助（待评估）**：在 LLM 判断前，先用关键词（如"知识库""资料""文档""有没有"）做预筛选
3. **二阶段检测（待评估）**：先判断是否与知识相关，再判断是否需要工具调用
4. **记录 bad case（待评估）**：收集更多误判案例，用作 few-shot examples
