## 四、技术决策

**当前技术栈**：FastAPI + DeepSeek API + PostgreSQL + MinIO + Milvus + BGE + LangGraph + Docker + uv/Python 3.12

| 组件 | 选择 | 要点 |
|------|------|------|
| LLM | DeepSeek API | 便宜、好用、OpenAI 兼容 |
| 后端 | FastAPI | 用户熟悉 |
| 前端 | 单文件 HTML/CSS/JS | 零学习成本 |
| 向量数据库 | Milvus Standalone | 企业级，父子块检索 |
| 对象存储 | MinIO（S3 兼容） | 用户明确要求 |
| 结构化存储 | PostgreSQL 17 | 端口 15433 |
| Embedding | BGE-large-zh-v1.5（本地，1024维） | 开源免费 |
| 文档分块 | 语义分块 + 父子块 | 2048/128 + 512/64 |
| 检索方式 | BM25 + 向量 → RRF 融合 | 混合检索 |
| 重排序 | bge-reranker-large（默认开启） | |

**三存储分工**：
- MinIO (:9000) → 文件柜，存原始文件
- Milvus (:19530) → 搜索引擎，存向量+chunk文本
- PostgreSQL (:15433) → 档案卡片柜，存元数据+聊天记录+父子块映射

**刻意不用**：LangGraph（简单请求-响应时）、Redis（阶段6引入）、GraphRAG（个人文档量小）
